axios-mock-adapter

Add a mock API on top of your existing Axios instance - intercept requests and return fake data without touching any component code.

Section 1

What is axios-mock-adapter?

axios-mock-adapter is a library that intercepts every Axios HTTP request before it leaves the browser and returns a pre-programmed fake response instead. Your React components never know the difference - they still call http.get(), http.post(), etc., and get real-looking data back.

React component
calls http.get()
→
Axios
→
MockAdapter
intercepts here
→
Real server
(never reached)
Instead, MockAdapter returns your fake data back to the component
💡

Why use it?

Frontend and backend teams can work in parallel. You can build and demo the full UI before writing a single line of server code.

Section 4

Adding the Mock (2 steps)

  • Create the mock file - src/mock-api/tutorialApi.js

    Import your existing Axios instance and pass it to MockAdapter, then define handlers for each route.

src/mock-api/tutorialApi.js 
import MockAdapter from 'axios-mock-adapter';
import http from '../http';

// Wrap our axios instance. delayResponse simulates network latency (ms).
const mock = new MockAdapter(http, { delayResponse: 300 });

// In-memory "database"
let tutorials = [
    { id: 1, title: 'React Basics', description: 'Intro to components.' },
    { id: 2, title: 'State & Effects', description: 'useState and useEffect.' },
];
let nextId = 3;

// ── GET /tutorial  (with optional ?search= query) ──
mock.onGet(/\/tutorial(\?.*)?$/).reply((config) => {
    const url = new URL(config.url, 'http://localhost');
    const search = url.searchParams.get('search')?.toLowerCase() ?? '';
    const result = search
        ? tutorials.filter(t =>
            t.title.toLowerCase().includes(search) ||
            t.description.toLowerCase().includes(search))
        : tutorials;
    return [200, result];
});

// ── GET /tutorial/:id ──
mock.onGet(/\/tutorial\/\d+/).reply((config) => {
    const id = parseInt(config.url.split('/').pop());
    const tutorial = tutorials.find(t => t.id === id);
    return tutorial ? [200, tutorial] : [404, { message: 'Not found' }];
});

// ── POST /tutorial ──
mock.onPost('/tutorial').reply((config) => {
    const body = JSON.parse(config.data);
    const newTutorial = { id: nextId++, ...body, createdAt: new Date().toISOString() };
    tutorials.push(newTutorial);
    return [201, newTutorial];
});

// ── PUT /tutorial/:id ──
mock.onPut(/\/tutorial\/\d+/).reply((config) => {
    const id = parseInt(config.url.split('/').pop());
    const body = JSON.parse(config.data);
    const index = tutorials.findIndex(t => t.id === id);
    if (index === -1) return [404, { message: 'Not found' }];
    tutorials[index] = { ...tutorials[index], ...body };
    return [200, tutorials[index]];
});

// ── DELETE /tutorial/:id ──
mock.onDelete(/\/tutorial\/\d+/).reply((config) => {
    const id = parseInt(config.url.split('/').pop());
    const index = tutorials.findIndex(t => t.id === id);
    if (index === -1) return [404, { message: 'Not found' }];
    tutorials.splice(index, 1);
    return [200, { message: 'Deleted successfully' }];
});

export default mock;
  • Import the mock file in src/main.jsx

    Import it once at the app entry point - before anything else renders. This ensures every Axios call is intercepted.

src/main.jsx 
import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';
import './mock-api/tutorialApi'; // <-- must be imported here so it registers before any request

ReactDOM.createRoot(document.getElementById('root')).render(
    <App />
);
Section 5

The Five Route Methods

MockAdapter mirrors the standard HTTP methods used in a REST API:

GET
onGet(url)

Fetch a list or a single item

POST
onPost(url)

Create a new item

PUT
onPut(url)

Replace / update an item

DELETE
onDelete(url)

Remove an item

PATCH
onPatch(url)

Partial update of an item

Every method takes a URL (string or RegExp) and chains .reply():

reply() syntax 
// Static reply: [statusCode, responseBody]
mock.onGet('/ping').reply(200, { message: 'pong' });

// Dynamic reply: callback receives the request config object
mock.onGet('/items').reply((config) => {
    // config.url    - the full URL string
    // config.data   - request body (string, parse with JSON.parse)
    // config.params - query params object (if passed via { params: {...} })
    return [200, [ {id:1, name:'foo'} ]];
});
Section 6

URL Matching - String vs RegExp

You can match routes with an exact string or a regular expression:

url matching examples 
// Exact string - matches only POST /tutorial
mock.onPost('/tutorial').reply(201, newItem);

// RegExp - matches GET /tutorial/1, /tutorial/42, etc.
mock.onGet(/\/tutorial\/\d+/).reply(handler);

// RegExp with optional query string - matches:
//   GET /tutorial
//   GET /tutorial?search=react
mock.onGet(/\/tutorial(\?.*)?$/).reply(handler);
✅

Tip: order matters - unless your regex anchors prevent overlap

MockAdapter matches routes top-to-bottom. Normally you'd put more specific routes first, but if your patterns use anchors like $, order doesn't matter. For example, /\/tutorial(\?.*)?$/ ends with $ so it will never match /tutorial/1 - meaning the /tutorial/:id handler can safely come after it without being shadowed.

Section 7

Reading Request Data in the Handler

The config object passed to your callback contains everything about the incoming request:

reading config 
// --- Reading a URL path parameter ---
mock.onGet(/\/tutorial\/\d+/).reply((config) => {
    const id = parseInt(config.url.split('/').pop()); // "/tutorial/5" → 5
    ...
});

// --- Reading query params ---
mock.onGet(/\/tutorial/).reply((config) => {
    const url    = new URL(config.url, 'http://localhost');
    const search = url.searchParams.get('search') ?? '';
    ...
});

// --- Reading the request body (POST/PUT) ---
mock.onPost('/tutorial').reply((config) => {
    const body = JSON.parse(config.data); // config.data is a JSON string
    ...
});
Section 8

Managing In-Memory Data

The mock "database" is just a plain JavaScript array declared at the top of tutorialApi.js. Every handler reads from and writes to this array, giving you full CRUD without a real database.

in-memory CRUD pattern 
let items = [
    { id: 1, name: 'Alpha' },
    { id: 2, name: 'Beta'  },
];
let nextId = 3;

// CREATE  →  push a new object, return 201
const newItem = { id: nextId++, ...body };
items.push(newItem);
return [201, newItem];

// READ    →  find by id
const item = items.find(i => i.id === id);

// UPDATE  →  find index, spread-merge, replace
const idx = items.findIndex(i => i.id === id);
items[idx] = { ...items[idx], ...body };

// DELETE  →  splice the array
items.splice(idx, 1);
⚠️

Data resets on page refresh

Because the array lives in JS memory, all changes are lost when the page reloads. This is expected for development - when you switch to a real server, data persists in the database.

Section 9

How React Components Use the Mock

Components call http (your Axios instance) exactly as they would with a real server. MockAdapter intercepts invisibly - no component code changes are needed when you later switch to a real backend.

pages/Tutorials.jsx (simplified) 
import { useEffect, useState } from 'react';
import http from '../http';

function Tutorials() {
    const [tutorials, setTutorials] = useState([]);

    useEffect(() => {
        // This looks exactly like a real API call.
        // MockAdapter intercepts it and returns our fake data.
        http.get('/tutorial').then(res => setTutorials(res.data));
    }, []);

    return (
        <ul>
            {tutorials.map(t => <li key={t.id}>{t.title}</li>)}
        </ul>
    );
}
Section 10

Simulating Errors

One of the biggest advantages of a mock is being able to test error states without breaking the real server:

error simulation 
// Return a 404 when item not found
const item = data.find(t => t.id === id);
if (!item) return [404, { message: 'Not found' }];

// Simulate a 500 server error
mock.onGet('/broken').reply(500, { message: 'Internal Server Error' });

// Simulate a network timeout (no response at all)
mock.onGet('/slow').timeout();

// Simulate a network error (connection refused)
mock.onGet('/offline').networkError();
Section 11

Switching to a Real API

When your backend is ready, simply stop importing mock-api/tutorialApi.js. All your component code stays unchanged.

src/main.jsx 
// Remove or comment out this one line to use the real server:
// import './mock-api/tutorialApi';

import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';

ReactDOM.createRoot(document.getElementById('root')).render(<App />);
✅

Zero component changes needed

Because components only import http.js and not tutorialApi.js directly, removing the mock import is the only change required to go live.

Section 12

Quick Reference

MockAdapter constructor options

new MockAdapter(axiosInstance, {
    delayResponse: 300,  // artificial delay in ms (simulates real network)
    onNoMatch: 'throwException' // throw if a URL has no handler (useful for debugging)
})

reply() return value

Always return a 2-element (or 3-element) array:

return [ statusCode, responseBody ];
return [ statusCode, responseBody, headers ]; // optional headers object

// Examples:
return [200, { id: 1, title: 'React' }];
return [201, newItem];
return [404, { message: 'Not found' }];
