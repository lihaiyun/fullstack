Testing Your REST API with curl

Windows CMD Student Guide — Tutorial API on http://localhost:3001

Before You Start
  • 1 Start the server. Open a terminal in the server/ folder and run npm run dev. You should see Server running on port 3001.
  • 2 Open a second CMD window. Press Win + R, type cmd, and press Enter. Keep it open while the server runs in the first window.
  • 3 Verify curl is available. Run the command below — you should see a version number, not an error. 
curl --version
✓ curl is built into Windows 10 and 11. If the command is not found, update Windows or download curl from curl.se.
1 — Create a Tutorial (POST)
POST

Create a new tutorial

/tutorial

This sends a JSON body to the server. The -H flag sets the content type so the server knows to parse the body as JSON. The -d flag provides the data.

Command
curl -X POST http://localhost:3001/tutorial -H "Content-Type: application/json" -d "{\"title\": \"My First Tutorial\", \"description\": \"This is a description of my first tutorial.\"}"
Expected Response
{"id": 1, "title": "My First Tutorial", "description": "This is a description of my first tutorial."}
ⓘ In CMD, use \" to include double quotes inside a double-quoted string. Do not use single quotes — CMD does not support them for quoting.
⚠ Validation rules: title must be 3–100 characters and description must be 3–500 characters. Both fields are required. A missing or too-short field returns 400 Bad Request.
201 Created 400 Bad Request
2 — Get All Tutorials (GET)
GET

List all tutorials

/tutorial

A plain GET request — no flags needed beyond the URL. The server returns a JSON array of every tutorial in the database.

Command
curl http://localhost:3001/tutorial
Expected Response
[{"id": 1, "title": "My First Tutorial", "description": "This is a description of my first tutorial."}]

Add a search query parameter to filter by keyword. Wrap the URL in quotes to prevent CMD from misreading the & or ? characters.

With Search Query
curl "http://localhost:3001/tutorial?search=first"
Expected Response
[{"id": 1, "title": "My First Tutorial", "description": "This is a description of my first tutorial."}]
200 OK
3 — Get a Tutorial by ID (GET)
GET

Get one tutorial

/tutorial/:id

Replace 1 at the end of the URL with the id value returned when you created the tutorial.

Command
curl http://localhost:3001/tutorial/1
Expected Response
{"id": 1, "title": "My First Tutorial", "description": "This is a description of my first tutorial."}
⚠ If the ID does not exist, the server returns 404 Not Found with an error message.
200 OK 404 Not Found
4 — Update a Tutorial (PUT)
PUT

Update an existing tutorial

/tutorial/:id

Same structure as POST — include the ID in the URL path. Both title and description are optional; omit either one to leave it unchanged.

Command
curl -X PUT http://localhost:3001/tutorial/1 -H "Content-Type: application/json" -d "{\"title\": \"Updated Title\", \"description\": \"Updated description text.\"}"
Expected Response
{"id": 1, "title": "Updated Title", "description": "Updated description text."}
200 OK 404 Not Found 400 Bad Request
5 — Delete a Tutorial (DELETE)
DELETE

Delete a tutorial

/tutorial/:id

No body needed — just set the method to DELETE and include the ID in the URL.

Command
curl -X DELETE http://localhost:3001/tutorial/1
Expected Response
{"message": "Tutorial deleted successfully."}
✓ Run curl http://localhost:3001/tutorial/1 again after deleting — you should get a 404, confirming the record is gone.
200 OK 404 Not Found
Quick Reference
Flag What it does Example
-X Sets the HTTP method -X POST
-H Adds a request header -H "Content-Type: application/json"
-d Sends data in the request body -d "{\"key\": \"value\"}"
-v Verbose — shows full request & response headers curl -v http://localhost:3001/tutorial
